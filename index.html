Prework Study Guide

HTML

  • The head element: contains information about the webpage.
  • The body element: represents the visible content shown to the user.

CSS

  • CSS: is a programming language used to define how webpage content is presented to users.
  • Inline CSS: is used to style an element directly in the HTML file
  • Internal CSS style sheet: contains CSS rules for the webpage using the style element in the head section of the HTML file.
  • External CSS style sheet: allows one to keep all their CSS rules in a separate file that is linked to the HTML file.
  • Seperation of concerns: is a term that means to separate code into different files based on use
  • A Selector: in a CSS rule defines the element or attributes to which the rules, or declarations, will apply
  • Declarations: in a CSS rule contain the property we want to apply and and the value of the property
  • A Margin: property indicates how much space we want around the outside of an element.
  • The padding property adds space around the content inside an element.
  • Class attributes: allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector. Then the class attribute can be assigned to the element in the HTML, enabling us to freely choose which elements will be assigned the CSS rule.
  • Two selectors can be affected by one rule by using a comma to seperate the two selectors.
  • The * symbol used as a selector, applies a rule to all the elements visible on the page.
  • A period preceding a selector will designate it to be a class selector and any HTML element assigned to the class will gain these property assignments from the CSS.

Git

  • git status: checks what branch we are currently on.
  • git checkout: used to switch to different branches when followed by which branch you wish to change to.
  • git pull origin: Pulls the remote branch into the local branch by running this command followed by which branch you wish to pull from.
  • git checkout -b branch-name: creates a new branch and also switches to it.
  • git add -A: stages changes made from local branch.
  • git commit -m “message”: commit message that describes the changes we have made.
  • git push origin: pushes up your changes from local to the remote repository when command is followed by the branch desired to be pushed.

JavaScript

  • JavaScript: a powerful programming language that can add interactivity to a website.
  • API: Application Programming Interface.
  • Variables: are containers that store values. Start by declaring a variable with the let keyword, followed by the name you give to the variable.
  • An operator: is a mathematical symbol that produces a result based on two values (or variables).
  • Conditionals: are code structures used to test if an expression returns true or not. A very common form of conditionals is the if...else statement.
  • Functions: are a way of packaging functionality that you wish to reuse.
  • Arguments: bits of data that functions need to do their job.
  • Events: real interactivity on a website requires event handlers. These are code structures that listen for activity in the browser, and run code in response.
  • const: keyword allows us to create variables that can't be reassigned a value.
  • =: assignment operator symbol assigns specific data that we want stored in our variable.
  • Control flow: is the order in which a computer executes code in a script.
  • if: when using the if statement, code will only execute if the statement meets a condition, or is true.
